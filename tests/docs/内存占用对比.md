# 内存占用对比

本文记录当前构建在同一台 macOS 机器上的内存占用。旧测试记录已删除，本文件只保留本轮重新采样结果。

## 环境

- 时间：2026-06-04 11:18 CST
- 系统：macOS 26.5，Darwin 25.5.0，arm64
- CPU：Apple M5
- 构建目录：`build/probes-opengl-glfw`
- 构建配置：`Release`、`EUI_RENDER_BACKEND=opengl`、`EUI_WINDOW_BACKEND=glfw`、`EUI_BUILD_TEST_FIXTURES=ON`、`EUI_DEPS_MODE=bundled`
- 采样方法：进程启动后等待 3 秒，使用 `ps -o rss= -o %cpu=` 每秒采样 6 次；同时使用 `vmmap -summary` 记录内存构成。

## 测试目标

| 目标 | 窗口尺寸 | 含义 |
| --- | --- | --- |
| `window_only_probe` | 800x600 | 只初始化 GLFW 并创建 OpenGL-capable 窗口，不创建 EUI RenderBackend。 |
| `render_context_probe` | 800x600 | 创建窗口并初始化 EUI RenderBackend / OpenGL context，不执行 clear/present。 |
| `render_clear_probe` | 800x600 | 创建窗口、初始化 RenderBackend，并持续 clear/present。 |
| `empty_window` | 800x600 | 走完整 AppRunner / DSL Runtime / RenderBackend，但 `compose()` 不创建 UI 元素。 |
| `animation_240fps` | 1280x720 | 真实 DSL 动画压力场景，目标 240 FPS。 |

## 结果

| 目标 | RSS 平均 | RSS 范围 | Physical footprint | Peak footprint | IOAccelerator graphics | IOSurface | MALLOC resident | MALLOC allocated | mapped file | owned unmapped graphics |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `window_only_probe` | 85.4 MiB | 85.4-85.5 MiB | 47.0 MiB | 81.5 MiB | 0.9 MiB | 14.8 MiB | 11.7 MiB | 6.6 MiB | 14.7 MiB | 15.1 MiB |
| `render_context_probe` | 85.5 MiB | 85.4-85.6 MiB | 47.0 MiB | 81.4 MiB | 0.9 MiB | 14.8 MiB | 11.7 MiB | 6.6 MiB | 14.8 MiB | 15.1 MiB |
| `render_clear_probe` | 85.7 MiB | 85.7-85.7 MiB | 98.2 MiB | 98.3 MiB | 5.4 MiB | 29.4 MiB | 11.6 MiB | 6.7 MiB | 14.8 MiB | 41.1 MiB |
| `empty_window` | 86.0 MiB | 86.0-86.0 MiB | 69.6 MiB | 103.9 MiB | 8.3 MiB | 29.4 MiB | 11.7 MiB | 6.7 MiB | 14.7 MiB | 15.1 MiB |
| `animation_240fps` | 103.9 MiB | 103.5-104.0 MiB | 164.7 MiB | 176.4 MiB | 24.2 MiB | 69.6 MiB | 18.9 MiB | 14.0 MiB | 16.2 MiB | 33.6 MiB |

## 结论

- `window_only_probe` 和 `render_context_probe` 在统一 800x600 后几乎一致，说明单纯初始化 EUI RenderBackend / OpenGL context 对 RSS 和普通 heap 的增量很小。
- `render_clear_probe` 的 RSS 仍只有约 85.7 MiB，但 `Physical footprint` 升到约 98.2 MiB，主要来自 clear/present 后出现的 drawable、IOSurface 和 driver graphics 资源。
- `empty_window` 走完整 EUI app/runtime 后，RSS 仍与 probe 接近；普通 heap allocated 约 6.7 MiB，说明空 DSL Runtime 本身不是大内存来源。
- `animation_240fps` 的 RSS 升到约 103.9 MiB，普通 heap allocated 约 14.0 MiB，增量主要来自真实 DSL 动画场景中的文本、图元、atlas、渲染资源和更大的 1280x720 surface 压力。
- 在 macOS 上，RSS 会包含大量系统 framework、窗口系统和图形栈常驻页；比较 EUI 自身成本时，应同时看 `Physical footprint` 和 `MALLOC allocated`。

## 复测命令

```sh

  window_only_probe render_context_probe render_clear_probe empty_window animation_240fps --parallel

build/probes-opengl-glfw/render_context_probe --manual
build/probes-opengl-glfw/render_clear_probe --manual
build/probes-opengl-glfw/empty_window
build/probes-opengl-glfw/animation_240fps
```
